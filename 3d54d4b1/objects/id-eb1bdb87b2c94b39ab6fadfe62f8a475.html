Reference data refers to a set of predefined, static information that provides context or categorisation for business activities, systems, or processes. It is typically used across various operations to ensure consistency, standardisation, and efficient data management. Unlike transactional data, which changes frequently (such as sales figures or inventory levels), reference data is relatively stable over time and provides the foundational building blocks for processing and interpreting other types of data. The reference data set currently in use is primarily focused on the areas of water and lighting. However, it is important to note that this set is not exhaustive. As operations grow and evolve, the reference data will need to be extended to cover other areas, incorporating additional code lists and categories specific to those new fields. This future expansion will ensure that all relevant operational areas are accurately captured and standardised across the organisation

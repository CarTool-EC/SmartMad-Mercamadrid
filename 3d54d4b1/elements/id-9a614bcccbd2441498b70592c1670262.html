Scorpio ()
Description: Scorpio is an NGSI-LD Broker that allows managing and requesting context information. Context Producers can manage their context – creating, updating, appending and deleting context information. Context Consumers can request the context information they require, either identifying the entity or discover relevant entities by providing the entity type, possibly filtering according to property value, existing relationship and/or geographic scope provided as a GeoJSON feature. Publisher: NEC Corporation Source: https://scorpio.rtfd.io/ Last modification: 2024-02-12 Identifier: http://data.europa.eu/dr8/egovera/ScorpioApplicationService LOST view: TVA-Data Space Enablers [Motivation] EIRA concept: eira:SolutionBuildingBlock Realises: http://data.europa.eu/dr8/DigitalSolutionApplicationService Title: Scorpio Application Service EIF Layer: TechnicalApplication Additional information: A Scorpio Broker in a distributed setting can then discover the Context Sources that may have information for answering a request based on their registrations, request and aggregate the information from different Context Sources and provide it to the requesting Context Consumer. In a federated setting the Context Source can itself be an NGSI-LD Broker. Federations can be used to combine information from multiple providers that want to (partially) share information. An important difference is then typically in the granularity of the registration, e.g. “I have information about entities of entity type building within a geographic area”, instead of “I have information about Building X” Additional identifier: http://data.europa.eu/dr8/egovera/ScorpioApplicationService Example: Scorpio implements NGSI-LD, which is the evolution of NGSI-v2 and previous versions going back to the OMA NGSI context interfaces. This standardized version is based on the NGSI-LD specification published by the ETSI Industry Specification Group on Context Information Management.
